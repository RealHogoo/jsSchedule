# 프로젝트 멤버 기능 문서

## 목적

프로젝트 멤버는 프로젝트에 실제 참여하는 사용자를 관리하는 기능이다.
조직 관리가 아니라 프로젝트 참여자 운영에 초점을 둔다.

## 기본 원칙

- 프로젝트 멤버는 사용자 단위로 저장한다.
- 팀 또는 부서는 기준 마스터가 아니라 검색 및 선택 보조 정보로 본다.
- 프로젝트 참여는 프로젝트별 스냅샷으로 관리한다.

## 화면 방향

- 프로젝트 내부 전용 페이지로 분리한다.
- 테이블 중심으로 참여자 조회, 추가, 복사, 수정 기능을 제공한다.

## 주요 화면

- 멤버 목록
- 멤버 추가
- 다른 프로젝트 멤버 불러오기

## 멤버 목록

- 사용자명
- 역할
- 투입기간
- 투입률
- 상태

## 멤버 추가 방식

- 개별 사용자 직접 추가
- 기존 프로젝트 멤버 구성 복사

## 복사 원칙

- 복사는 참조가 아니라 스냅샷으로 처리한다.
- 원본 프로젝트 인원 변경이 현재 프로젝트에 자동 반영되면 안 된다.
- 복사 후 현재 프로젝트에서 독립적으로 수정 가능해야 한다.

## 기능 원칙

- 노드 복사와 멤버 복사는 분리한다.
- 노드 담당자 배정은 별도 기능으로 둔다.
- 프로젝트 멤버 구성 복사는 프로젝트 레벨 기능으로 제공한다.

## 고려 사항

- `팀 단위 추가`가 필요하면 실제 저장은 사용자별 행으로 생성한다.
- 역할, 투입기간, 투입률을 추후 확장 가능한 구조로 두는 것이 좋다.
