# 대시보드 기능 문서

## 목적

대시보드는 프로젝트 현재 상태를 요약해서 보여주는 운영 화면이다.
상세 작업 처리보다 상황 인지와 우선순위 판단에 초점을 둔다.

## 화면 방향

- 프로젝트 상세 홈의 상단 요약 영역 또는 별도 대시보드 페이지로 구성한다.
- 수치 카드, 상태 분포, 지연 현황, 예정 마일스톤 중심으로 배치한다.

## 주요 정보

- 전체 진행률
- 상태별 태스크 수
- 지연 태스크 수
- 금주 또는 금월 예정 일정
- 참여자별 작업 현황

## 로그 기준

- 집계성 작업은 `JOB_START`, `JOB_END`, `JOB_FAIL` 로그 규약을 따른다.
- 대시보드 요약 데이터는 필요 시 별도 집계 로직을 둘 수 있다.

## UI 원칙

- 운영자가 한눈에 상태를 파악할 수 있어야 한다.
- 숫자와 상태를 빠르게 읽을 수 있게 구성한다.
- 상세 확인이 필요한 영역은 해당 기능 화면으로 링크한다.

## 고려 사항

- 대시보드는 상세 화면을 대체하지 않는다.
- 과도한 차트보다 즉시 해석 가능한 요약 정보가 우선이다.
