# 캘린더 기능 문서

## 목적

캘린더는 프로젝트 일정과 태스크 일정을 시간 축 기준으로 보여주는 화면이다.
작업 목록과는 다른 관점에서 일정 충돌과 집중 구간을 파악하는 데 사용한다.

## 화면 방향

- 캘린더는 전용 페이지로 분리한다.
- 월간 보기와 주간 보기를 기본으로 제공한다.
- 프로젝트, 담당자, 상태 필터를 상단에 둔다.

## 주요 화면

- 월간 캘린더
- 주간 캘린더
- 일정 상세 팝업

## 표현 기준

- 프로젝트 일정
- 마일스톤 일정
- 태스크 일정

## UI 원칙

- 일정 생성보다 조회와 충돌 확인을 우선한다.
- 일정 클릭 시 상세 팝업 또는 상세 페이지로 연결한다.
- 캘린더에서 직접 수정하는 기능은 최소화한다.

## 고려 사항

- 동일 기간 다수 태스크가 겹칠 수 있으므로 필터가 중요하다.
- 지연 태스크와 예정 마일스톤을 구분해서 표현할 수 있어야 한다.
- 프로젝트 전체 관점과 개인 담당자 관점을 모두 지원하는 것이 좋다.
