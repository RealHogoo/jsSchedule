# 태스크 기능 문서

## 목적

태스크는 프로젝트 내 실제 작업 수행 단위다.
프로젝트 진행률, 담당자 배정, 일정 관리, 노드 구조 관리의 기준 엔티티로 사용한다.

## 화면 방향

- 기본 화면은 목록 중심으로 구성한다.
- 프로젝트 선택 이후 태스크 목록과 태스크 편집을 같은 작업 맥락에서 처리한다.
- 별도 화면 전환보다 인라인 작업영역 교체를 우선한다.

## 주요 화면

- 태스크 목록
- 태스크 등록 및 수정
- 태스크별 노드 작업영역

## 태스크 목록

- 프로젝트 기준으로 조회한다.
- 주요 컬럼은 태스크명, 상태, 우선순위, 담당자, 기간, 진행률로 구성한다.
- 사용자는 목록에서 태스크를 선택하고 같은 화면에서 바로 수정할 수 있어야 한다.

## 태스크 등록 및 수정

- 태스크 등록은 목록 화면 하단 작업영역을 폼으로 전환해 처리한다.
- 태스크 수정도 동일한 위치에서 처리한다.
- 저장 후에는 목록과 프로젝트 요약 값이 함께 갱신되어야 한다.
- 프로젝트를 바꾸면 이전 태스크 편집 상태는 정리한다.

## 태스크별 노드 작업영역

- 태스크를 선택하면 해당 태스크의 노드 트리를 조회한다.
- 노드 작업영역은 태스크 화면 내부에 포함한다.
- 사용자는 같은 화면에서 노드를 등록, 수정, 삭제, 이동할 수 있어야 한다.
- 노드에는 태스크 타입별 통계 항목 값이 연결된다.

## 배정 정책

- 담당자는 현재 프로젝트 멤버 중에서 선택한다.
- 태스크 담당자는 태스크 저장 시점에 검증한다.
- 태스크와 노드의 담당 정책은 분리할 수 있으나, 1차에서는 태스크 중심으로 단순화한다.

## 구조 정책

- 태스크는 프로젝트 하위에서만 존재한다.
- 노드는 태스크 하위 트리 구조로 존재한다.
- 노드 상세 구조와 통계 항목 설계는 [node.md](D:/MSA_project/ADMIN_project/schedule-service/docs/node/node.md)를 기준으로 한다.

## 고려 사항

- 지연 태스크, 완료 예정 태스크, 미확정 태스크를 빠르게 찾을 수 있어야 한다.
- 태스크 등록과 수정은 사용자의 작업 맥락을 유지해야 한다.
- 트리 구조가 있더라도 태스크 화면 자체는 과도하게 분리하지 않는다.
