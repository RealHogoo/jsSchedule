# 프로젝트 기능 문서

## 목적

프로젝트는 `schedule-service`의 최상위 업무 관리 단위다.
프로젝트는 일정, 태스크, 마일스톤, 참여자, 대시보드의 기준 컨텍스트가 된다.

## 화면 방향

- 일반 웹페이지 구조를 사용한다.
- 기본 레이아웃은 `헤더 + 좌측 내비게이션 + 본문`으로 구성한다.
- 프로젝트 상세는 단일 화면에 모든 기능을 몰아넣지 않고 기능별 페이지로 분리한다.

## 주요 화면

- 프로젝트 목록
- 프로젝트 상세 홈
- 프로젝트 설정

## 프로젝트 목록

- 기본 표현은 카드보다 테이블 중심이 적합하다.
- 주요 컬럼은 프로젝트명, 상태, 기간, PM, 참여인원, 진행률, 최근 수정일로 구성한다.
- 상태, 기간, 담당자, 프로젝트 타입 기준 필터를 제공한다.

## 프로젝트 상세 홈

- 상단에 프로젝트 기본 정보와 상태를 노출한다.
- 본문에는 진행률, 지연 작업 수, 예정 마일스톤, 최근 활동, 참여자 요약을 배치한다.
- 상세 홈은 운영자가 현재 상태를 빠르게 파악하는 요약 화면으로 사용한다.

## 프로젝트 설정

- 프로젝트 기본 정보 수정
- 프로젝트 상태 변경
- 프로젝트 타입 선택
- 프로젝트 멤버 초기 구성

## 데이터 방향

- 프로젝트는 공통 코어 구조를 유지한다.
- 프로젝트 타입은 공통 필드와 선택 필드 조합으로 확장한다.
- 타입별로 완전 분리 테이블을 두기보다 공통 테이블 중심으로 설계한다.

## 고려 사항

- 프로젝트는 조직 마스터가 아니라 업무 운영 단위로 본다.
- 다른 프로젝트의 멤버 구성을 현재 프로젝트로 복사하는 기능을 제공할 수 있다.
- 프로젝트 복사 기능은 구조 복사와 멤버 복사를 분리해서 다룬다.
