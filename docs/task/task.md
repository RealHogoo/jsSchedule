# 태스크 기능 문서

## 목적

태스크는 프로젝트 단위의 실제 작업을 관리하는 기본 엔티티입니다.
일정, 담당자, 진행률, 상위 태스크, WBS 색상, 상세 설명을 함께 관리합니다.

## 화면 구성

- 기본 진입은 `project.html` 또는 `task-form.html`입니다.
- 프로젝트를 선택하면 같은 화면 안에서 태스크 목록과 편집 폼을 전환합니다.
- 별도 WBS 보드는 `/wbs.html`에서 제공합니다.

## 태스크 목록

- 프로젝트 기준으로 조회합니다.
- 목록에서 태스크를 선택하면 같은 화면에서 바로 수정할 수 있습니다.
- 상위 태스크가 있는 경우 들여쓰기 형태로 트리 순서를 보여줍니다.
- 일정이 없는 태스크도 목록에는 표시합니다.

## 저장 항목

- `task_title`
- `task_status`
- `priority`
- `assignee_user_id`
- `start_date`
- `due_date`
- `actual_start_date`
- `actual_end_date`
- `progress_rate`
- `description`
- `parent_task_id`
- `wbs_color`

## 상위 태스크 규칙

- 태스크는 같은 프로젝트 안에서만 상위 태스크를 가질 수 있습니다.
- 자기 자신을 상위 태스크로 지정할 수 없습니다.
- 하위 태스크를 다시 부모로 지정하는 순환 구조는 허용하지 않습니다.
- 루트 포함 최대 4단계까지만 허용합니다.

## WBS 색상

- 태스크 폼에서 `WBS 색상`을 직접 지정할 수 있습니다.
- 값은 `#RRGGBB` 형식의 6자리 HEX 색상만 허용합니다.
- 값이 비어 있거나 잘못된 경우 기본값 `#0F766E`를 사용합니다.
- 이 색상은 `/wbs.html` 막대 렌더링에 사용됩니다.

## WBS 화면

- 상단에서 프로젝트를 선택합니다.
- 좌측에는 태스크명과 기간을 보여줍니다.
- 우측에는 프로젝트 시작일~종료일 범위 안에서 태스크 기간 막대를 그립니다.
- 일정이 없는 태스크는 목록에는 보이지만 차트 막대는 그리지 않습니다.
- 모바일에서는 좌측 목록 대신 차트와 하단 오버레이 상세만 보여줍니다.

## 관련 API

- `POST /task/list.json`
- `POST /task/detail.json`
- `POST /task/save.json`
