# 태스크 기능 문서

## 목적

태스크는 프로젝트 내 실제 작업 수행 단위다.
프로젝트 진행률, 캘린더, 대시보드의 핵심 데이터 소스가 된다.

## 화면 방향

- 기본 화면은 테이블 중심으로 구성한다.
- 필요 시 트리 또는 칸반 뷰를 보조적으로 제공한다.
- 초기 메인 화면은 테이블 뷰를 우선한다.

## 주요 화면

- 태스크 목록
- 태스크 상세
- 태스크 일괄 수정

## 태스크 목록

- 프로젝트 기준으로 조회한다.
- 주요 컬럼은 태스크명, 상태, 우선순위, 담당자, 시작일, 종료일, 진행률, 상위 노드로 구성한다.
- 필터는 상태, 담당자, 기간, 마일스톤, 지연 여부를 지원한다.

## 태스크 상세

- 태스크 기본 정보
- 담당자 및 참여자
- 기간 및 진행률
- 관련 마일스톤
- 메모 또는 활동 이력

## 구조 원칙

- 노드 복사는 구조 복사로만 다룬다.
- 사용자 등록은 복사 기능과 분리된 배정 기능으로 다룬다.
- 태스크 담당자는 현재 프로젝트 멤버 중에서 선택한다.

## 배정 원칙

- 프로젝트 멤버가 먼저 구성되어야 한다.
- 태스크 담당자 일괄 적용은 별도 기능으로 제공한다.
- 노드 복사 시 담당자 자동 복사는 기본값으로 사용하지 않는다.

## 고려 사항

- 지연 태스크, 완료 예정 태스크, 미배정 태스크를 빠르게 찾을 수 있어야 한다.
- 트리 구조가 있더라도 상세 수정은 폼 중심이 적합하다.
